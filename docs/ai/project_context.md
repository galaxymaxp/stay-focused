# Stay Focused — Project Context

Author: galaxymaxp omgraythekid@gmail.com
Last Updated: 2026-04-29

## Product North Star

Stay Focused is now a **schedule-first student workflow app over Canvas**.

The core promise is simple: students should open Stay Focused and immediately know **what they should do next**, based on a smart schedule generated from their academic workload and personal availability.

## Core Experience

The main product surface should center on an auto-generated study schedule built from:

- user free-time windows
- Canvas modules
- tasks
- deadlines
- priority
- estimated completion/review time

This schedule is the operational layer of the product: it translates scattered academic requirements into actionable study blocks.

## Role of Calendar

Calendar remains important, but its role is specific:

- Calendar is the source of deadlines and events.
- Calendar data feeds the scheduler.
- Calendar is **not** the primary “what do I do now?” surface.

## Supporting Systems

Reviewers, Deep Learn, quizzes, and Do drafts should all support scheduled blocks.

When a user starts a schedule block, the app should open the right execution context automatically, including the relevant:

- review material
- source file
- quiz
- task draft
- Do Now helper

## UX Direction (Persistent)

- reduce clutter
- prioritize “what should I do next?”
- hide secondary details behind progressive disclosure
- keep urgent actions visible
- preserve the soft-glow yellow accent
- preserve the modern 2026 full-width responsive style

## Guardrail

Do not change behavior by default during documentation work. Product direction updates should be reflected in docs first, then implemented intentionally.
