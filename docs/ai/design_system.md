# Stay Focused — Design System Direction (Schedule-First)

Author: galaxymaxp omgraythekid@gmail.com
Last Updated: 2026-04-29

## Product UI Intent

The interface should feel focused, calm, and academically professional while continuously answering: **“What should I do next?”**

## Visual Principles

- Reduce clutter and visual noise.
- Keep urgent actions obvious at first glance.
- Use progressive disclosure to defer secondary metadata.
- Preserve the soft-glow yellow accent language.
- Preserve the modern 2026 full-width responsive style.

## Information Hierarchy Rules

1. **Primary layer:** next scheduled block + immediate next actions.
2. **Secondary layer:** rationale/context (course, deadline, priority, time estimate).
3. **Tertiary layer:** deep details, configuration, and historical/reference content.

If information does not help the user start or continue their current block, it should usually be secondary or tertiary.

## Schedule-Block Interaction Standard

When a user starts a schedule block, UI should prioritize direct execution:

- Open the matching work surface quickly.
- Surface only the minimum context needed to begin.
- Keep support actions nearby (review source, open module, view due date).
- Avoid full-page context switches unless required.

## Calendar Presentation Rule

Calendar should remain available and trustworthy for date context, but should not dominate primary task execution flows.

Calendar UI should emphasize its feeder role to scheduling:

- deadlines/events in
- scheduled focus blocks out

## Responsive Layout Direction

- Keep full-width workspace behavior on modern large screens.
- Avoid narrow center-column lockups for primary workflow pages.
- Maintain readable density with clear panel hierarchy and spacing rhythm.

## Tone of Interface Components

- Clear and directive over decorative.
- Helpful and task-forward over exploratory.
- Structured and lightweight over dashboard clutter.
